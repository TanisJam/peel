**SUGGESTION**:
1. **`bin[peel]` cleanup warning** during `npm publish`: npm normalized `./dist/index.js` to `dist/index.js`. Harmless but cosmetic; fix in a follow-up by setting `"bin": { "peel": "dist/index.js" }` directly to suppress the warning. Or run `npm pkg fix` once.
2. **`src/version.ts` drift risk** — mitigated for 0.1.0 via manual sync. Follow-up change: read package.json's version at build time via tsup `define` so `version.ts` stays auto-synced.
3. **GitHub Actions release automation** (`changesets/action`) — add as a follow-up so future releases don't need a manual `npm publish` step.
4. **Coverage tooling** (`@vitest/coverage-v8`) — recurring suggestion; would let us enforce per-file thresholds.
5. **Re-deploy workflow note**: this `release/0.1.0` branch had a transient npm `404 Not Found` due to expired auth — confirming `npm whoami` BEFORE running `prepublishOnly` would catch the same class of issue earlier. Add to a release runbook in the README dev section.
